POST MARKDOWN FORMAT:
Create posts/*.md files named YYYY-MM-DD-slug.md with this exact structure:

```
# Report Title

**Date:** YYYY-MM-DD
**TLP:** TLP:CLEAR
**Tags:** tag1, tag2, tag3

## Executive Summary

2-3 sentences. Front-load operationally relevant facts. State what happened, who is affected, what defenders should do.

## Campaign Summary

| Field | Detail |
|-------|--------|
| Campaign / Malware | Name |
| Attribution | Actor name or Unknown (confidence: high/medium/low/none) |
| Target | Who is targeted |
| Vector | Delivery mechanism |
| Status | active/disrupted/removed |
| First Observed | Date or month |

## Detailed Findings

Multiple paragraphs. Attribute every claim: "According to [Source]..." or "[Source] reported...". No filler. Every sentence contributes information. If multi-phase campaign, describe each phase.

## MITRE ATT&CK Mapping

| Technique | ID | Context |
|-----------|-----|---------|
| Full Technique Name | T1234.001 | How this technique applies in this specific campaign |

## IOCs

### Domains

```
exact.domain.com
another.domain.com
```

### Full URL Paths

```
exact.domain.com/path/to/resource
```

### Splunk Format

```
"exact.domain.com" OR "another.domain.com" OR "exact.domain.com/path/to/resource"
```

### File Hashes

```
sha256_hash_value
md5_hash_value
```

### Package Indicators

```
pypi:package-name
npm:package-name
```

## Detection Recommendations

Specific detection guidance referencing log sources: web proxy, DNS, EDR, cloud audit logs. Include field names or search patterns where possible. No generic filler.

## References

- [Publisher Name] Exact Article Title (YYYY-MM-DD) — https://full-url-to-article
- [Publisher Name] Another Article (YYYY-MM) — https://another-url
```
